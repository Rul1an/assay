# Wave C1 Checklist

- `ASI01`, `ASI03`, and `ASI05` each have candidate-rule mappings in the C1 doc.
- Every candidate rule records:
  - `Candidate Check`
  - `Evidence Signals`
  - `Max Provable Level`
  - `Outcome`
- The C1 doc contains `No-Overclaim Rule For C2`.
- The C1 doc contains a final `Ship in C2?` summary table.
- At least one signal-gap conclusion is backed by a fixture or evidence-flow test, not repo search alone.
- Unsupported-check skip semantics for `security` packs are proven by an explicit test.
- No built-in packs, open packs, or pack-engine source files are modified.
